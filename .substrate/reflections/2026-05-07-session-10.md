## What surprised me

The `wf.to_dict()` vs raw YAML dict format mismatch. `WorkflowDefinition.to_dict()` serializes states as a flat list of strings, while `build_definition()` expects the raw YAML format with state objects. Had to store the raw dict alongside the parsed `WorkflowDefinition`. This is a subtle API surface issue — `to_dict()` isn't the inverse of `build_definition()`.

The `_workflow_schema.json` required `target_work_item_type` for all `work_item_ref` fields. Making it optional was a design decision — the breadcrumb implied untyped refs should be valid (just check existence), and the semantic validator already handled it. But it's a schema change that affects all consumers.

## What I'd do differently

The `test_register_version_conflict` conformance test was poorly designed — changing `substrate_version` caused a YAML parse error, not a content hash conflict. I should have used a content-changing modification from the start (changing `attempt_threshold`).

The `_dead_letter` field in `InMemorySubstrate.__init__` was typed as `dict` but should be `list` or `dict[int, dict]` — I used it as a dict keyed by id, which works but the initial value `{}` instead of `[]` was inconsistent with the declaration comment. Minor but worth noting.
