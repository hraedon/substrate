## On the project

Substrate is in a mature state. The event-sourced model, `_contract.py` single-source-of-truth extraction, and the signing scheme are well-designed. The `_contract.py` refactor (RFC-062) was the right structural move — it centralized validation and decision logic. But the conformance story between InMemory and Postgres is the project's weakest structural point. The InMemory backend has drifted from being a faithful test double; it reimplements sequencing, idempotency, event construction, and claim/state management independently, and the drift shows (6 of Kimi's 14 findings are InMemory conformance gaps, and 2 of my 3 supplementary findings are too).

The breadcrumb discipline is strong. Every defect gets a numbered file with frontmatter, severity, and fix suggestion. The resolved/ archive goes back 127 items. This is unusual and valuable.

The test suite (300+ tests) is good for happy-path coverage but the property-based conformance tests (`test_property_conformance.py`) don't assert on `event_seq` or `last_event_seq`, don't exercise idempotency paths, and don't send adversarial inputs for error-code equivalence. These gaps explain how BC-115 (off-by-one seq) and BC-116 (missing idempotency) survived.

This was a review-only session — no code changes beyond breadcrumb files and README updates. ## On what remains

1. **BC-128, BC-129, BC-130** — three new open breadcrumbs I filed. BC-129 is probably the most urgent since it subsumes BC-116's fix scope.
2. **EventStore extraction** — the single highest-return structural change. Extracts seq allocation, idempotency, and event construction from both backends into shared code. Kills BC-115, BC-116, BC-129 structurally.
3. **Push validation to contract** — shared pre-check layer for `_validate_actor_kind`, `_validate_event_id`, `_validate_not_before_delta` at every public method entry point. Kills BC-118, BC-125 structurally.
4. **Harden property tests** — add `event_seq`/`last_event_seq` assertions, idempotency-exercising strategies, adversarial input strategies for error-code matching. Without this, conformance drift remains silent.
5. **Duplicate table in README** — lines 61-79 of `breadcrumbs/README.md` duplicate the resolved table entries. Should be deduplicated.
