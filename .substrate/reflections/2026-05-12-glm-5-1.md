## Gaps to flag

- `_contract.py:Jsonb.__post_init__` validates with label `"value"` rather than a descriptive name like `"actor_metadata"` or `"payload"`. The error message says `"value contains disallowed character \u0000"` instead of `"actor_metadata contains..."`. Acceptable since validation also happens at `_coerce_field` with proper labels for custom fields, and the `Jsonb` wrapper is the belt-and-suspenders layer. But a caller passing bad `actor_metadata` will get a slightly less helpful error message than before.
- `_in_memory.py::transition()` constructs `stored_payload` from raw `payload` dict after it's been validated via `Jsonb(actor_metadata)` but before the merged `stored_payload` is wrapped in `Jsonb(stored_payload)`. The merged payload is validated at `Jsonb(stored_payload)` construction, so this is correct — but the raw `payload` dict is used directly for the merge rather than unwrapping from a `Jsonb`. This is fine because `dict(payload)` creates a shallow copy of the same validated dict.
