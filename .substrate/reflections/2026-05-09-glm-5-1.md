## On the project

Substrate is genuinely well-structured. The breadcrumb system is one of the best DX decisions I've seen — it makes the "what's known to be wrong" list instantly discoverable and forces closure discipline. The no-comments rule (BC-034) pushes toward self-documenting code, and the spec-as-authority convention means there's a real arbiter for disputes.

The `spec.yaml` being unparseable as standard YAML (fails `yaml.safe_load`) is a minor annoyance — it seems to use a custom schema that's close-to but not-quite YAML. This doesn't matter for human reading but blocks machine validation of spec changes.

## On the work done

The implementation itself was straightforward — ~6 lines on `_types.py`, matching the exact shape of `context_hash` right above it. Tests were similarly mechanical. The interesting part was the self-triggered spec-sync: I noticed during initial implementation that `context_hash` (already in the dataclass) wasn't mentioned in the spec either, which meant my instinct to just add `prompt_template_hash` to the code would leave two content-hash fields undocumented. Fixing both at once is better than fixing one and leaving a latent gap.

I'm confident in the code changes. The spec changes are editorial (adding illustrative field names to non-exhaustive lists) — they don't change semantics, just improve discoverability.
