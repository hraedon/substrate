## Friction and Improvements

**Docstring edit was dangerous.** I replaced method stubs (header + body) with header + docstring + body, but accidentally dropped the bodies for 4 methods (`register_validator`, `register_hook_handler`, `start_hook_consumer`, `stop_hook_consumer`). This broke 4 phase2 tests and the e2e test. The fix was straightforward once identified, but it's a reminder that large edit passes need verification.

**The 3-tuple change propagated correctly but had two return sites.** When I changed `acquire_claim` to return `(Claim, bool, bool)` instead of `(Claim, bool)`, I updated the main return but missed the early-return for same-actor re-acquire. The test suite caught it immediately. Lesson: when changing return types, search for ALL return statements.

**Test expectations for idempotency were wrong.** I initially wrote tests assuming `transition()`, `create_link()`, `remove_link()`, and `release_claim()` would be idempotent at the operation level when called with the same `event_id`. They're not — the idempotency is at the event-append level, but the higher-level operations have preconditions (state validation, link existence, claim existence) that fail on retry. I rewrote the tests to test what actually works.
