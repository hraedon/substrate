## On the project

Substrate is in strong shape. The adversarial review cycle (BC-064 through BC-138) has been thorough and the codebase reflects it — every serious conformance gap between backends has been systematically identified and fixed. The `_contract.py` extraction (RFC-062) works well as shared validation logic, and the adversarial conformance test (`test_adversarial_error_code_equivalence`) provides a meaningful CI gate.

The InMemory backend is still the weakest structural point. Every deep scan finds conformance gaps because it reimplements Postgres sequencing, idempotency, event construction, replay error handling, hook queue management, and read-events ordering independently. The fixes are correct but each one is a band-aid over the structural problem: two independent implementations of the same semantics. BC-128 (EventStore protocol extraction) remains the single highest-return change.

## On the work done

The three-agent parallel scan was effective. One agent found 13 issues in `_in_memory.py`, another found 11 in `_contract.py`, the third found 9 in `_events.py`/`_replay.py`. The high-signal findings clustered around: InMemory replay error handling (BC-133, the biggest gap), InMemory hook queue data loss on requeue (BC-132), and the Postgres UniqueViolation gap (BC-134).

I'm most confident in BC-132 (requeue data loss), BC-133 (replay error handling), and BC-134 (UniqueViolation) — these are straightforward, test-verified fixes. BC-135 (timezone normalization) is a defensive fix; the real-world risk is low since all internal callers pass aware datetimes, but the guard is correct. BC-136 (read_events sort) and BC-137 (hook queue IDs) are clear conformance fixes.

The breadcrumbs README deduplication was tedious but necessary — the table had grown through multiple sessions' edits with inconsistent merging.

## On what remains

1. **BC-128 — EventStore protocol** — still the top structural priority. Would kill BC-115, BC-116, BC-129, BC-132, BC-133, BC-136, BC-137 structurally by replacing dict-based reimplementation with shared code.
2. **Property test hardening** — the adversarial conformance test is good but `_compare_state` doesn't assert event_seq/last_event_seq, and no strategy exercises idempotency paths. The HypothesisDeprecationWarning should be fixed too.
3. **`_states_match` and `claim_expires_at`** — currently intentionally excluded. If heartbeats ever emit events, this comparison should be enabled. Documented in BC-130 resolution.
4. **Design items from scan not filed as breadcrumbs:**
   - `resolve_heartbeat` treats missing `expires_at` as non-expired (should probably error)
   - Inconsistent expired-claim handling between `validate_release` (allows) and `resolve_heartbeat` (rejects)
   - `last_event_at` hardcoded to NULL in replay output table (derivable from last event timestamp)
   - InMemory `acquire_claim` calls `_check_escalation` on extend (harmless divergence)
5. **Pending directory cleanup** — `breadcrumbs/pending/README.md` is stale, only file remaining.
